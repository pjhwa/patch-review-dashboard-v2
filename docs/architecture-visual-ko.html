1
전처리 (Python)
Python 스크립트가 벤더별 원시 데이터를 정규화된 PreprocessedPatch 객체로 변환합니다. 각 제품은 고유한 preprocessingScript와 preprocessedPatchMapper() 함수를 보유합니다. 변환 결과는 SQLite의 PreprocessedPatch 테이블에 저장됩니다.
Python 3 RawPatch → PreprocessedPatch Prisma upsert
2
AI 검토 루프 (OpenClaw) — "AI 검토 루프" 탭에서 상세 내용 확인
전처리된 패치를 5개 배치 단위로 나누어 OpenClaw AI 에이전트에 순차적으로 전달합니다 (openclaw agent --agent main --json --timeout 1800). AI는 각 패치의 중요도 (Critical / Important / Moderate / Low)와 결정 (Done / Exclude / Pending)을 평가하고 한국어 설명을 작성합니다. JSON 출력은 Zod 스키마로 검증되며, 실패 시 에러 내용을 프롬프트에 주입하여 최대 2회 재시도합니다. 각 배치 성공 후 즉시 DB에 저장(mid-pipeline 영속성)됩니다. API Rate Limit 감지 시 플래그 파일을 기록하고 즉시 중단합니다.
배치 크기: 5개 OpenClaw AI (타임아웃 1800초) Zod 검증 최대 2회 재시도 Rate Limit 이어하기
3
DB 저장 (Prisma)
검증된 AI 검토 결과를 ReviewedPatch 테이블에 Prisma upsert로 저장합니다. 고유 키는 [vendor, issueId] 복합 인덱스입니다. 기존 레코드가 있으면 업데이트하고, 없으면 신규 삽입합니다. 실행 로그는 PipelineRun 레코드에 감사 이력으로 기록됩니다.
Prisma upsert ReviewedPatch 테이블 PipelineRun 로그
4
패스스루 안전망
AI가 처리하지 못한 패치(타임아웃, 배치 실패 등)를 기본값으로 자동 삽입합니다. 기본 설정: criticality="Important", decision="Pending". 데이터 손실 제로를 보장하며, 제품별로 passthrough.enabled로 활성화 여부를 설정합니다.
데이터 손실 제로 보장 기본값: Important/Pending
5
분기별 자동 아카이브
9개 제품 전체가 검토를 완료하면 분기별 스냅샷을 자동으로 생성합니다. ReviewedPatch 테이블 전체를 patches.json과 metadata.json으로 내보냅니다. 저장 경로: ~/.openclaw/workspace/skills/patch-review/quarterly-archive/{분기}/. 각 제품 완료 후 POST /api/archive/quarterly/auto-check를 호출하여 트리거합니다.
분기별 자동 실행 JSON 스냅샷 감사 이력 보존
RAG 제외 전략 — 전 제품 통합 적용
프롬프트 인젝션 전략 (9개 전 제품)
적용 제품: 전 제품(9개)
os/linux/에 위치한 공유 query_rag.py가 user_exclusion_feedback.json에서 관리자가 등록한 과거 배제 이력을 조회하여 CRITICAL INSTRUCTION 블록으로 AI 프롬프트에 주입. AI가 유사 패치를 자동으로 배제 처리하도록 유도.
파일 숨김 전략 (비Linux 제품, 프롬프트 인젝션과 병행)
적용 제품: Windows, Ceph, MariaDB, SQL Server, PostgreSQL
AI 호출 전 normalized/ 디렉토리와 패치 JSON 파일을 임시로 숨겨 OpenClaw가 이전 리뷰 데이터를 파일 도구로 읽지 못하게 방지. AI 루프 종료 후 복원. 프롬프트 인젝션과 함께 type: 'both'로 적용.
전략 유형: prompt-injection — Linux 3종(Red Hat, Oracle, Ubuntu) & VMware vSphere  |  both — Windows, Ceph, MariaDB, SQL Server, PostgreSQL (파일 숨김 + 프롬프트 인젝션)
