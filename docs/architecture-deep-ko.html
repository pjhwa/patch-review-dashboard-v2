🔒 withOpenClawLock — 동시성 제어

// 디렉토리 뮤텍스 — 동시 AI 호출 방지 // 잠금 파일: /tmp/openclaw_execution.lock async function withOpenClawLock<T>(fn: () => Promise<T>): Promise<T> { const lock = await acquireLock('/tmp/openclaw_execution.lock'); try { return await fn(); } finally { releaseLock(lock); } }

⚡ OpenClaw AI 에이전트

# 전체 호출 시그니처 openclaw agent:main \ --json \ --timeout 1800 \ --session-id {unique-per-batch} \ -m {prompt-string}
  • ▸ --json: JSON 전용 출력 모드 강제
  • ▸ --timeout 1800: 배치당 30분 타임아웃
  • ▸ --session-id {unique}: 배치 간 오염 방지를 위해 배치별 고유값
  • ▸ -m {prompt}: RAG 컨텍스트를 포함한 전체 프롬프트 문자열
  • ▸ queue.ts 워커에 의해 자식 프로세스로 실행됨
  • ▸ withOpenClawLock (디렉토리 뮤텍스)으로 보호됨

👤 관리자 검토 & 피드백 루프

📋 관리자 검토 워크플로우 — 단계별

1
파이프라인 실행
관리자가 대시보드에서 파이프라인을 트리거합니다. BullMQ가 작업을 처리합니다. SSE를 통해 실시간 로그 확인. AI가 할당한 심각도와 결정으로 패치가 ReviewedPatch 테이블에 나타납니다.
2
대시보드에서 패치 검토
관리자가 제품 상세 페이지를 확인합니다. 필터 테이블 (심각도, 결정, 컴포넌트별). 각 패치에 표시: IssueID, 컴포넌트, 버전, 심각도, AI 결정, 한국어 설명, 이유.
3
제외 피드백 제공
관리자가 제외 패턴 패치를 표시합니다. 피드백 텍스트는 향후 실행에서 유사한 패치가 제외되어야 하는 이유를 설명합니다. UserFeedback 테이블과 user_exclusion_feedback.json에 저장됩니다.
4
피드백 → ChromaDB
user_exclusion_feedback.json이 ChromaDB 벡터 스토어에 임베딩됩니다. 향후 RAG 쿼리가 의미적으로 유사한 과거 제외 사례를 검색합니다. 순환 피드백 루프가 시간이 지남에 따라 개선됩니다.
5
확정 — CSV 생성
관리자가 확정 버튼을 클릭합니다. POST /api/pipeline/{product}/finalize가 final_approved_patches_{product}.csv를 생성합니다. 비Linux 제품은 한국어 Excel 텍스트 호환성을 위해 UTF-8 BOM (csvBOM=true) 사용.
